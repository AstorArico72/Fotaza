Guía de Markdown y formato

Ésto funciona para subidas, comentarios, y perfiles de usuario.

En Fotaza, es necesario Markdown para que el texto se vea bien, y no esté todo junto. También es admisible el marcado HTML.
El intérprete Markdown que usamos (Marked) sigue las especificaciones GitHub Flavored Markdown (GFM) 0.29, y CommonMark 0.30, ambas licenciadas bajo CC-BY-SA, al igual que el Markdown original de Gruber.

  • Saltos de línea

    Termina la línea con dos espacios antes de presionar "Enter".

  • Línea separadora

    Escribe tres guiones (---) al comenzar la línea, pero no escribas nada más en ésa línea.

  • Encabezados

    Comienza una línea con uno o más "hash" (#), así: 

                    
# Encabezado H1
## Encabezado H2
...
###### Encabezado H6

  • Negrita e itálica

    Encierra tu texto entre dos asteriscos (*) para el texto en negrita. Encierralo entre un * para el texto en itálica. 

                        
*Itálica*
**Negrita**
***Negrita mas itálica***

  • Listas

    Pon un guión (-) o un signo más (+) - pero no los mezcles - delante de cada ítem para listas desordenadas. Usa números o letras seguidas de un punto para listas ordenadas. La lista es creada automáticamente. No olvides poner un espacio entre los símbolos y los ítems. 

                        
Lista desordenada
- Item 1
- Item 2
- Item 3

Lista ordenada
1. Item
2. Item
3. Item

I. Item
II. Item
III. Item
Ten en cuenta que la lista se ordena sola según qué ítem está primero.

  • Enlaces externos

    Encierra la dirección del enlace entre flechas (<>) para un enlace automático. Para enlaces con texto, coloca el texto entre corchetes, seguido de la dirección entre paréntesis. También puedes guardar los enlaces en variables al comienzo del texto y usarlos después, así: 

                        
Autoenlace
<http://www.example.com>

Enlace dentro de un párrafo
Lorem ipsum dolor sit amet consectetur adipisicing elit.  
[Enlace 1](http://www.example.net "Título opcional del enlace aquí") Temporibus eum laboriosam, enim aut reprehenderit magnam necessitatibus! Repellendus ad eaque, in

Éso se transforma en:
<p>
Lorem ipsum dolor sit amet consectetur adipisicing elit.<br>
<a href="http://www.example.net" title="Título opcional del enlace aquí">Enlace 1</a>Temporibus eum laboriosam, enim aut reprehenderit magnam necessitatibus! Repellendus ad eaque, in
</p>

Enlace guardado en referencia, que puede reusarse dentro del texto
[Enlace 2]: http://www.example.org "Título opcional del enlace aquí"

Usando el enlace guardado
Lorem ipsum dolor sit amet consectetur adipisicing elit. Temporibus eum laboriosam, enim aut reprehenderit magnam necessitatibus! [Enlace 2]

Éso se transforma en:
<p>
Lorem ipsum dolor sit amet consectetur adipisicing elit. Temporibus eum laboriosam, enim aut reprehenderit magnam necessitatibus! <a href="http://www.example.org" title="Título opcional del enlace aquí"> Enlace 2 </a>
</p>

  • Imágenes externas

    Igual que la sección de enlaces (pero excluyendo los autoenlaces). Sólo añade un signo de exclamación delante de los corchetes. 

                        
Imágen simple
![Texto alt de la imágen](http://www.example.net/image.png "Título opcional")
Éso se transforma en:
<img src="http://www.example.net/image.png" alt="Texto alt de la imágen" title="Título opcional">

Imágen sin texto alt
![](http://www.example.net/image.png)

Éso se transforma en:
<img src="http://www.example.net/image.png">

Imágen guardada como referencia
[Imágen 1]: http://www.example.net/image.png "Título opcional"

Usando la imágen guardada
Lorem ipsum dolor sit amet consectetur adipisicing elit. Temporibus eum laboriosam, enim aut reprehenderit magnam necessitatibus ![Imágen 1]
Nótese cómo no hay un "!" al guardar la referencia, pero sí hay un "!" al llamar a la referencia.

  • Tablas

    Encierra el texto de las columnas entre "tubos" ("|"), pero tiene que haber un espacio entre el "tubo" y el texto, así: | foobar | 

                        
Nota: La primera fila siempre es interpretada como <th> en lugar de <td>
| Columna 1 | Columna 2 |
| --- | --- |
| foobar | bazbim |
| lunes | martes |
La segunda fila separa el encabezado del cuerpo de la tabla.
Las etiquetas <table>, <thead>, y <tbody> se crean automáticamente.

  • Código y texto de ancho fijo

    Encierra el texto entre tres acentos graves (```) o tres tildes (~~~), ni más ni menos, para un bloque de código. Encierra el texto entre un par de acentos para una sección de código. 

                        
~~~
Bloque de código encerrado entre tildes.
No es necesario terminar la línea con doble espacio para un salto de línea en los bloques de código.
~~~

```
Bloque de código encerrado entre acentos.
```

Foo `Código indentado en un párrafo` bar
Puede haber un espacio separando el acento del código.
`` Código que contiene un acento ` como ése debe abrirse y cerrarse con dos acentos ``

  • Texto tachado

    Encierra el texto entre una o dos tildes (~). 

                        
~~Fotaza es basura~~
~Muere~

  • Videos y audio externos, iframes, y estilo.

    Éso no está cubierto por la sintaxis Markdown, pero puedes insertar las etiquetas HTML correspondientes también. Revisa alguna guía de HTML.

    Etiquetas prohibidas por motivos de seguridad: <script>, <form> y todo tipo de <input>.
    Etiquetas no permitidas por interferir con la disposición: <footer>, <aside>.
